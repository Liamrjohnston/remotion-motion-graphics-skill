# Approved reference mechanisms

Select by story mechanism. Inspect the bundled MP4 and contact sheet before coding.

## `openmontage-hook`

Use for many fragmented tools/stages collapsing into one coherent production system.

- Opens tight on crowded browser tabs and partial interface—not an isolated title card.
- Pulls back to reveal real workflow surfaces sharing one world.
- Surfaces physically collapse toward an authentic terminal/product center.
- Finished timeline, audio, captions, and output surfaces appear as consequences.
- Path: congested detail → context reveal → transformation hold → finished-system payoff.

## `openmontage-claude-pipeline`

Use for a request moving through real production stages.

- One connected pipeline, not disconnected scenes.
- Camera follows causality from input through agent work into output surfaces.
- Product identity and production objects remain visible.
- Each destination is a distinct step; travel bridges them.

## `tsenta-apply-faster`

Use for slow manual work replaced by automation.

- Opens inside an authentic form at readable task scale.
- Typing and elapsed-time evidence establish the old way.
- Camera reveals accumulated work, then travels into authentic product actions.
- Cursor clicks cause visible state changes; a wave of results is the payoff.
- Path: manual action → accumulated-cost reveal → automated actions → scale payoff.

## `tsenta-one-click-wall`

Use for one action creating a large result.

- Opens tight on one authentic product card and cursor action.
- Click causes the state change.
- Accelerating pullback reveals a coherent wall of related results.
- Scale is spatial, not an arbitrary chart/headline.

## `kickbacks-same-run-editor`

Use for an AI/code/tool action inside the authentic working surface.

- Editor is the world, not a decorative card.
- Camera pans across the exact code/action changing.
- Runtime state and payoff remain attached to that surface.
- Detail, state, and result are shown in working context.

## `smartlead-prewarmed-burn`

Use for a product action causing a metric/system state consequence.

- Faithful product UI supplies identity.
- Cursor launches the campaign.
- Health, risk, chart, and sent count change as direct consequences.
- Camera is restrained because authentic UI and causality carry the story.

## Selection rule

- fragmentation → consolidation: `openmontage-hook`
- sequential production workflow: `openmontage-claude-pipeline`
- manual labor → automation: `tsenta-apply-faster`
- one action → massive scale: `tsenta-one-click-wall`
- action inside real editor/tool: `kickbacks-same-run-editor`
- product action → metric consequence: `smartlead-prewarmed-burn`

If none match, stop and request a reference or plan approval. Never invent a generic fallback.
